Experiment 08

Parameter Passing Mechanisms

C++ Programming

üéØ Aim of the Experiment

Write a program to demonstrate parameter passing mechanism in C++

üíª Program

C++ 
#include <iostream>
using namespace std;

void callByValue(int x) {
    x = x + 10;
    cout << "Inside callByValue: " << x << endl;
}

void callByReference(int &x) {
    x = x + 10;
    cout << "Inside callByReference: " << x << endl;
}

void callByPointer(int *x) {
    *x = *x + 10;
    cout << "Inside callByPointer: " << *x << endl;
}

int main() {
    int num = 5;
    
    cout << "Original value: " << num << endl;
    
    callByValue(num);
    cout << "After callByValue: " << num << endl;
    
    callByReference(num);
    cout << "After callByReference: " << num << endl;
    
    callByPointer(&num);
    cout << "After callByPointer: " << num << endl;
    
    return 0;
}

üìã Sample Output

Output 
Original value: 5
Inside callByValue: 15
After callByValue: 5
Inside callByReference: 15
After callByReference: 15
Inside callByPointer: 25
After callByPointer: 25

üìö Key Concepts Demonstrated

  • Call by Value: Function receives a copy of the argument; original value unchanged
  • Call by Reference: Function receives reference to original variable; changes affect original
  • Call by Pointer: Function receives memory address; dereference to modify original
  • Parameter Modification: Understanding which methods preserve vs modify original data
  • Memory Management: Efficient parameter passing for large data structures
  • Function Design: Choosing appropriate passing mechanism based on requirements

‚úÖ Conclusion

Parameter passing demonstrates call by value preserves original while reference and pointer modify original values. Understanding these mechanisms is crucial for efficient function design and memory management in C++ programming.
